Architecture (résumé)

  • UI: JavaFX + FXML + CSS
  • DB: MySQL via JDBC + pool HikariCP
  • Rôles: ADMIN / SECRETAIRE / MEDECIN / PATIENT
  • Accès secrétaire: secrétaire lié à un ou plusieurs médecins (table SecretaireMedecin)
  • Fonctions: Patients, médecins, spécialités, disponibilités, rendez-vous, dossiers médicaux, ordonnances

Rôle des dossiers et fichiers (résumé)

src/main/java/com/example/gestion_medicale/
  Main.java                : point d’entrée JavaFX + init DB + hooks de fermeture
  DatabaseConnection.java   : HikariCP + getConnection()
  DatabaseInitializer.java  : exécute init_database.sql (Collectors/Streams)
  SessionManager.java       : session utilisateur + currentPatientId
  AppExecutors.java         : ExecutorService (threads) pour tâches DB

src/main/java/com/example/gestion_medicale/controllers/
  LoginController.java           : login + récupération PatientCompte si rôle PATIENT
  MainController.java            : navigation + boutons selon rôle
  UserController.java            : CRUD utilisateurs + liaison PATIENT + liaison SECRETAIRE↔MEDECIN (multi-select)
  DoctorController.java          : gestion médecins (sans champ mot de passe dans l’écran)
  PatientController.java         : gestion patients + création auto dossier médical
  RendezVousController.java      : gestion RDV + filtrage secrétaire + Task/ExecutorService + HashMap
  DossierMedicalController.java  : dossiers + filtrage secrétaire/médecin/patient + liste ordonnances
  OrdonnanceController.java      : médecin crée ordonnance; patient consulte

src/main/java/com/example/gestion_medicale/models/
  POJOs: User, Patient, Doctor, Specialite, Disponibilite, RendezVous, DossierMedical, Ordonnance

src/main/resources/com/example/gestion_medicale/
  login.fxml, MainView.fxml, *Management.fxml, styles.css

src/main/resources/
  database.properties, init_database.sql

Titres du cours → où c’est utilisé (dans ce projet)

  • Collections (List/Map):
    • ObservableList (List) — où: src/main/java/com/example/gestion_medicale/controllers/*Controller.java (ex: RendezVousController, DoctorController, PatientController, DossierMedicalController, OrdonnanceController, UserController). Pourquoi: stocker des lignes pour TableView/ComboBox (données dynamiques UI).
    • HashMap (Map) — où: src/main/java/com/example/gestion_medicale/controllers/RendezVousController.java. Pourquoi: indexer id → Patient et id → Doctor pour remplir rapidement les combos lors de la sélection d’un RDV (accès O(1)).
  • Streams / Collectors:
    • Collectors.joining + Arrays.stream — où: src/main/java/com/example/gestion_medicale/DatabaseInitializer.java. Pourquoi: lire init_database.sql et préparer/concaténer les lignes SQL en String, puis supprimer les commentaires avant exécution.
    • stream().filter().findFirst() — où: ex. src/main/java/com/example/gestion_medicale/controllers/DoctorController.java et .../DossierMedicalController.java. Pourquoi: retrouver un objet dans une liste (sélectionner une spécialité/Patient correspondant).
  • Threads (Multithreading):
    • ExecutorService (Thread Pool) — où: src/main/java/com/example/gestion_medicale/AppExecutors.java. Pourquoi: exécuter des tâches DB en arrière-plan sans bloquer l’UI.
    • JavaFX Task — où: src/main/java/com/example/gestion_medicale/controllers/RendezVousController.java. Pourquoi: charger Patients/Médecins/RDV en asynchrone (meilleure UX: interface fluide).
    • synchronized — où: src/main/java/com/example/gestion_medicale/SessionManager.java (getInstance()). Pourquoi: protéger l’accès au singleton si plusieurs threads l’utilisent.
  • JDBC:
    • PreparedStatement / ResultSet / try-with-resources — où: contrôleurs CRUD, ex: src/main/java/com/example/gestion_medicale/controllers/UserController.java, .../RendezVousController.java, .../DoctorController.java, .../PatientController.java, .../DossierMedicalController.java, .../OrdonnanceController.java. Pourquoi: exécuter des requêtes SQL paramétrées (sécurité) et fermer automatiquement les ressources.
    • Pool de connexions (HikariCP) — où: src/main/java/com/example/gestion_medicale/DatabaseConnection.java + config src/main/resources/database.properties. Pourquoi: performances (réutilisation connexions) et configuration externalisée.
