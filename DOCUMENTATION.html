🔧 2.2 module-info.java

Déclare les dépendances du module Java et les packages ouverts/exportés :

module com.example.gestion_medicale { // Accès aux composants JavaFX requires javafx.controls; requires javafx.fxml; // Accès à java.sql (Connection, PreparedStatement…) requires java.sql; // Accès à HikariCP pour le pool de connexions requires com.zaxxer.hikari; // Ouvre le package principal aux FXMLLoader de JavaFX opens com.example.gestion_medicale to javafx.fxml; // Ouvre controllers pour que FXML injecte les @FXML opens com.example.gestion_medicale.controllers to javafx.fxml; // Ouvre models pour PropertyValueFactory (TableView) opens com.example.gestion_medicale.models to javafx.fxml, javafx.base; // Exporte les packages publics exports com.example.gestion_medicale; exports com.example.gestion_medicale.controllers; exports com.example.gestion_medicale.models; }
💡 Pourquoi opens ? JavaFX utilise la réflexion pour injecter les champs @FXML et accéder aux propriétés des modèles. Sans opens, une InaccessibleObjectException serait levée.

3. Couche Base de Données

🔌 3.1 DatabaseConnection.java

Gère le pool de connexions HikariCP. C'est le seul point d'accès à la base de données dans toute l'application.

Méthodes principales :

Méthode Type retour Description
getConnection() Connection Retourne une connexion du pool (thread-safe)
close() void Ferme le pool HikariCP à la fin de l'app

Fonctionnement :

// 1. Lit les propriétés depuis database.properties Properties props = new Properties(); props.load(stream); // charge depuis le classpath // 2. Configure HikariCP HikariConfig config = new HikariConfig(); config.setJdbcUrl(props.getProperty("db.url")); config.setUsername(props.getProperty("db.username")); config.setPassword(props.getProperty("db.password")); config.setMaximumPoolSize(10); // max 10 connexions simultanées config.setConnectionTimeout(30000); // timeout 30s // 3. Création du DataSource (singleton) dataSource = new HikariDataSource(config); // 4. Utilisation dans les contrôleurs : try (Connection conn = DatabaseConnection.getConnection()) { // connexion automatiquement retournée au pool après le bloc }
💡 HikariCP est le pool de connexions le plus rapide pour Java. Il maintient des connexions ouvertes pour éviter le coût de reconnexion à chaque requête.

🏗 3.2 DatabaseInitializer.java

Exécuté au démarrage de l'application. Crée les tables si elles n'existent pas et insère les comptes par défaut.

Rôle :

  • Lit et exécute init_database.sql via JDBC
  • Crée la table users avec CREATE TABLE IF NOT EXISTS
  • Crée la table doctors avec CREATE TABLE IF NOT EXISTS
  • Insère l'administrateur par défaut si non existant : admin / admin123
  • Insère la secrétaire par défaut : secretaire / sec123

📄 3.3 init_database.sql

Script SQL exécuté par DatabaseInitializer. Définit le schéma complet de la base :

-- Table utilisateurs CREATE TABLE IF NOT EXISTS users ( id INT AUTO_INCREMENT PRIMARY KEY, username VARCHAR(100) UNIQUE NOT NULL, password VARCHAR(255) NOT NULL, role ENUM('admin', 'secretary', 'doctor') NOT NULL ); -- Table médecins (liée à users) CREATE TABLE IF NOT EXISTS doctors ( id INT AUTO_INCREMENT PRIMARY KEY, nom VARCHAR(100) NOT NULL, prenom VARCHAR(100) NOT NULL, specialite VARCHAR(100) NOT NULL, user_id INT, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL );

4. Modèles (Models)

👤 4.1 User.java

Représente un utilisateur de l'application. Utilisé par la TableView et les contrôleurs.

Attributs :

Attribut Type Description
id int Identifiant unique auto-incrémenté (clé primaire)
username String Nom d'utilisateur unique (login)
password String Mot de passe (stocké en clair, à hasher en prod)
role String Rôle : admin, secretary, ou doctor

Structure :

public class User { private int id; private String username, password, role; // Constructeur complet public User(int id, String username, String password, String role) { ... } // Getters et Setters pour chaque attribut public int getId() { return id; } public String getUsername() { return username; } public String getRole() { return role; } // ... etc }
💡 Pourquoi des getters ? La TableView JavaFX utilise PropertyValueFactory("username") qui appelle automatiquement getUsername() par réflexion.

prenom String Prénom du médecin
specialite String Spécialité médicale (ex: Cardiologie)

5. Point d'entrée — Main.java

🚀 5.1 Main.java

Classe principale qui démarre l'application JavaFX. Hérite de Application (obligatoire en JavaFX).

Cycle de vie :

public class Main extends Application { @Override public void start(Stage primaryStage) throws IOException { // ÉTAPE 1 : Initialise les tables MySQL au démarrage DatabaseInitializer.initialize(); // ÉTAPE 2 : Charge le FXML de login FXMLLoader loader = new FXMLLoader( Main.class.getResource("/com/.../login.fxml")); Scene scene = new Scene(loader.load()); // ÉTAPE 3 : Configure la fenêtre primaryStage.setTitle("Gestion Médicale – Connexion"); primaryStage.setWidth(520); // largeur 520px primaryStage.setHeight(480); // hauteur 480px primaryStage.setResizable(true); // redimensionnable primaryStage.centerOnScreen(); // centré à l'écran primaryStage.show(); } @Override public void stop() { // Appelé à la fermeture : libère le pool HikariCP DatabaseConnection.close(); } public static void main(String[] args) { launch(args); // démarre JavaFX Application Thread } }

Points importants :

  • start() : méthode principale, s'exécute sur le JavaFX Application Thread
  • stop() : garantit la fermeture propre du pool HikariCP
  • launch() : démarre le moteur JavaFX et appelle start()
  • La fenêtre démarre en 520×480px, centrée, redimensionnable

6. Contrôleurs — LoginController & MainController

🔐 6.1 LoginController.java

Gère l'authentification des utilisateurs. Lié à login.fxml.

Champs @FXML injectés :

Champ Type FXML Description
usernameField TextField Saisie du nom d'utilisateur
passwordField PasswordField Saisie du mot de passe (masqué)
messageLabel Label Affiche les erreurs en rouge
loginButton Button Bouton "Se connecter"

Méthode handleLogin() :

private void handleLogin() { // 1. Valide que les champs ne sont pas vides if (username.isEmpty() || password.isEmpty()) { showError(...); return; } // 2. Désactive le bouton pendant la tentative loginButton.setDisable(true); // 3. Authentifie contre la BDD User user = authenticate(username, password); // 4. Bloque le rôle "doctor" if ("doctor".equalsIgnoreCase(user.getRole())) { showError("❌ Accès refusé. Admins et secrétaires uniquement."); return; } // 5. Passe l'utilisateur à MainController (variable statique) MainController.setCurrentUser(user); // 6. Charge MainView.fxml et change la scène FXMLLoader loader = new FXMLLoader(..."MainView.fxml"); primaryStage.setScene(new Scene(loader.load())); primaryStage.setWidth(1150); // fenêtre principale plus grande primaryStage.setHeight(720); }

Requête SQL d'authentification :

SELECT id, username, role FROM users WHERE username = ? AND password = ?

🧭 6.2 MainController.java

Gère la navigation dans la fenêtre principale (sidebar + contenu dynamique).

Mécanisme de navigation :

// Variable statique pour passer l'utilisateur entre contrôleurs private static User currentUser; // Charge un FXML dans le contentArea private void loadPane(String fxmlFile) { FXMLLoader loader = new FXMLLoader(getClass().getResource(fxmlFile)); contentArea.getChildren().setAll(loader.load()); } // Boutons sidebar → showUsers(), showDoctors(), showDashboard() @FXML private void showUsers() { loadPane("UserManagement.fxml"); } @FXML private void showDoctors() { loadPane("DoctorManagement.fxml"); } // Déconnexion : recharge login.fxml @FXML private void handleLogout() { currentUser = null; // charge login.fxml sur la scène principale }

Statistiques du dashboard :

  • Requête SELECT COUNT(*) FROM users pour afficher le nb d'utilisateurs
  • Requête SELECT COUNT(*) FROM doctors pour le nb de médecins

👥 6.3 UserController.java

Implémente toutes les opérations CRUD sur les utilisateurs. Lié à UserManagement.fxml.

Champs @FXML :

Champ Type Rôle

Méthodes CRUD :

Méthode SQL Description
loadUsers() SELECT * FROM users Charge tous les utilisateurs dans la TableView
addUser() INSERT INTO users... Ajoute un nouvel utilisateur
updateUser() UPDATE users SET... Met à jour (password optionnel si vide)
deleteUser() DELETE FROM users Supprime après confirmation Alert
handleSearch() — Filtre la liste en mémoire (pas de SQL)

Initialisation :

@Override public void initialize(URL url, ResourceBundle rb) { // Peuple le ComboBox rôles (pas dans FXML pour éviter erreurs) roleCombo.getItems().addAll("admin", "secretary"); setupTable(); // configure colonnes + cellFactories loadUsers(); // charge données depuis MySQL clearForm(); // réinitialise le formulaire }

🩺 6.4 DoctorController.java

Même architecture que UserController, adapté pour les médecins.

Différences avec UserController :

  • userIdCombo : ComboBox peuplée avec les utilisateurs (id — username) depuis la BDD
  • Badge spécialité : couleur teal (#0891b2) au lieu des badges rôles
  • 4 champs : nom, prenom, specialite, user_id (vs username, password, role)

Requêtes SQL spécifiques :

-- Chargement médecins SELECT id, nom, prenom, specialite, user_id FROM doctors ORDER BY id DESC -- Ajout médecin INSERT INTO doctors (nom, prenom, specialite, user_id) VALUES (?, ?, ?, ?) -- Modification UPDATE doctors SET nom=?, prenom=?, specialite=?, user_id=? WHERE id=? -- Suppression DELETE FROM doctors WHERE id=?

10. Base de Données MySQL Railway

🗄 10.1 Schéma des tables

Table users :

CREATE TABLE IF NOT EXISTS users ( id INT AUTO_INCREMENT PRIMARY KEY, username VARCHAR(100) UNIQUE NOT NULL, -- unique, login password VARCHAR(255) NOT NULL, -- à hasher en prod role ENUM('admin','secretary','doctor') NOT NULL );

Table doctors :

CREATE TABLE IF NOT EXISTS doctors ( id INT AUTO_INCREMENT PRIMARY KEY, nom VARCHAR(100) NOT NULL, prenom VARCHAR(100) NOT NULL, specialite VARCHAR(100) NOT NULL, user_id INT, FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE SET NULL -- si user supprimé, user_id = NULL );

Relation entre tables :

users doctors ┌─────────────────────┐ ┌───────────────────────────────┐ │ id (PK) │◄────────│ user_id (FK, nullable) │ │ username │ │ id (PK) │ │ password │ │ nom │ │ role │ │ prenom │ └─────────────────────┘ │ specialite │ └───────────────────────────────┘ Un médecin peut être lié à un compte utilisateur (optionnel)
